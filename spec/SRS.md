# 四川麻将（血战到底）- 需求规格说明书 (SRS)

## 1. 引言

### 1.1 目的
本文档描述四川麻将（血战到底）Web 游戏的功能需求和非功能需求。

### 1.2 项目背景
开发一个基于 Web 的四川麻将游戏，支持本地 4 人对战（1 真人 + 3 AI）。

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| 血战到底 | 一家胡牌后，其他玩家继续游戏，直到流局或三家胡牌 |
| 定缺 | 游戏开始时选择一门花色，胡牌时手中不能有此花色 |
| 碰 | 手牌中有两张相同牌时，可碰别人打出的第三张 |
| 杠 | 手牌中有三张相同牌时，可杠别人打出的第四张（明杠）或自己摸到第四张（暗杠） |
| 点炮 | 打出的牌被别人胡了 |
| 自摸 | 自己摸牌后胡牌 |

## 2. 功能需求

### 2.1 游戏流程

```
[等待开始] → [定缺] → [游戏进行] → [结算]
```

#### 2.1.1 等待开始
- 显示游戏标题和开始按钮
- 点击开始游戏后发牌，显示手牌后进入定缺阶段

#### 2.1.2 定缺阶段
- **先展示手牌，玩家查看后再选择定缺**
- 显示各花色手牌数量统计
- **规则：手牌少于 3 张的花色不能被选为定缺**
- 玩家选择一门花色定缺
- AI 自动选择手牌最少的花色定缺（同样受 3 张限制）
- 所有玩家定缺完成后自动进入游戏

#### 2.1.3 游戏进行
- 庄家先出牌（14张牌）
- **弃牌区布局：打出的牌放在自己手牌前方（靠近牌桌中心）**
- 回合顺序：摸牌 → 打牌 → 其他玩家响应
- 响应优先级：胡 > 杠 > 碰
- 一家胡牌后，该玩家退出，其他玩家继续
- 流局或三家胡牌时游戏结束

#### 2.1.4 结算
- 显示所有玩家最终得分
- 显示排名
- 可重新开始游戏

### 2.2 核心规则

#### 2.2.1 牌型
- 使用 108 张牌（筒、条、万，各 1-9，每种 4 张）
- 无字牌、无花牌

#### 2.2.2 胡牌条件
- 必须缺一门（不能有定缺花色的牌）
- 牌型为 4 组刻子/顺子 + 1 对
- 刻子：3 张相同的牌
- 顺子：同花色连续 3 张牌

#### 2.2.3 计分规则（简化版）
- 平胡点炮：1 番
- 平胡自摸：2 番
- 杠牌不计分（MVP 简化）

### 2.3 玩家操作

#### 2.3.1 打牌
- 点击手牌即可打出
- 必须在倒计时内出牌

#### 2.3.2 碰
- 其他玩家打出可碰的牌时显示"碰"按钮
- 点击后形成碰牌组合（三张相同）

#### 2.3.3 杠
- 明杠：别人打出第四张时
- 暗杠：自己摸到第四张时
- 杠后摸一张牌

#### 2.3.4 胡
- 可胡别人的牌（点炮）
- 可胡自己摸的牌（自摸）
- 胡牌后显示"胡"标记

#### 2.3.5 过
- 选择不响应碰/杠/胡
- 超时自动过

### 2.4 AI 行为

#### 2.4.1 定缺策略
- 计算手牌中各花色数量
- 选择数量最少的花色定缺

#### 2.4.2 打牌策略
- 优先打定缺花色的牌
- 其次打孤张（无法组成顺子/刻子的牌）

#### 2.4.3 碰杠策略
- 有 70% 概率碰非定缺花色的牌
- 有 80% 概率杠非定缺花色的牌
- 能胡牌时优先胡

### 2.5 界面需求

#### 2.5.1 布局（欢乐麻将风格）
- **4 家牌在同一屏幕显示，无需滚动**
- **弃牌区位置**：打出的牌放在自己手牌前方（靠近牌桌中心）
- 牌桌中心只显示最后出牌和剩余张数
- 自己显示在底部
- 其他三家分别显示在左、上、右

#### 2.5.2 玩家区域布局（从外到内）
1. **玩家信息**（最外侧）：名称、分数、庄家标记、定缺标记
2. **手牌**（中间）：背面朝上（其他玩家）或正面朝上（自己）
3. **弃牌区**（最内侧，靠近牌桌）：打出的牌，按顺序排列

#### 2.5.2 信息显示
- 玩家名称、分数、庄家标记
- 定缺花色显示
- 听牌提示（可胡的牌高亮）
- 当前回合玩家高亮

#### 2.5.3 操作反馈
- 按钮：碰（绿）、杠（紫）、胡（红）、过（灰）
- 倒计时显示

## 3. 非功能需求

### 3.1 性能
- 页面加载时间 < 3 秒
- 游戏操作响应时间 < 100ms

### 3.2 兼容性
- 支持 Chrome、Firefox、Safari 最新版本
- 支持桌面端（优先）

### 3.3 可维护性
- 核心逻辑 100% 测试覆盖率
- 使用 TypeScript 类型检查
